Get started with MCP servers in E2B sandboxes.

<CodeGroup>
```typescript TypeScript
import Sandbox from 'e2b'

const sandbox = await Sandbox.Create({
    mcp: {
        duckduckgo: {},
    },
});

const mcpUrl = sandbox.getMcpUrl();
```

```python Python
from e2b import Sandbox

sandbox = await Sandbox.create(
    mcp={
        "duckduckgo": {},
    },
)

mcp_url = sandbox.get_mcp_url()
```
</CodeGroup>
## Accessing the MCP Gateway

The MCP gateway can be accessed from both outside and inside the sandbox, depending on your use case. Both require you to use the Authorization header.

### From Outside the Sandbox

You can access the MCP gateway from outside the sandbox using the URL and authentication token:

<CodeGroup>
```typescript TypeScript
import { Client } from '@modelcontextprotocol/sdk/client/index.js';
import { StreamableHTTPClientTransport } from '@modelcontextprotocol/sdk/client/transports';

const transport = new StreamableHTTPClientTransport(
    new URL(sandbox.betaGetMcpUrl()), 
    {
        requestInit: {
            headers: {
                'Authorization': `Bearer ${await sandbox.betaGetMcpToken()}`
            }
        }
    }
);

await client.connect(transport);

const tools = await client.listTools();
```

### From Inside the Sandbox

If you need to access the MCP gateway from within the sandbox itself, it's available at:
```
http://e2b.local:50005/mcp
```


## Limitations

- slow docker pull
- no custom mcp server support
- no oauth support
